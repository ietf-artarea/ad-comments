79	1.  Introduction

81	   The syntax for private-key information was originally described in
82	   [RFC5208] and later obsoleted by [RFC5958].  This document defines
83	   PKCS #8 content types for use with PrivateKeyInfo and
84	   EncryptedPrivateKeyInfo.

86	2.  Conventions and Definitions

88	   The key words "MUST", "MUST NOT", "REQUIRED", "SHALL", "SHALL NOT",
89	   "SHOULD", "SHOULD NOT", "RECOMMENDED", "NOT RECOMMENDED", "MAY", and
90	   "OPTIONAL" in this document are to be interpreted as described in
91	   BCP 14 [RFC2119] [RFC8174] when, and only when, they appear in all
92	   capitals, as shown here.

94	3.  Private-Key Information Content Types

96	   This section defines a content type for private-key information and
97	   encrypted private-key information.

99	   The PrivateKeyInfo content type is identified by the following object
100	   identifier:

102	   id-ct-privateKeyInfo OBJECT IDENTIFIER ::= { iso(1)
103	    member-body(2) us(840) rsadsi(113549) pkcs(1) pkcs-9(9)
104	    smime(16) ct(1) TBD1 }

106	   The EncryptedPrivateKeyInfo content type is identified by the
107	   following object identifier:

109	   id-ct-encrPrivateKeyInfo OBJECT IDENTIFIER ::= { iso(1)
110	    member-body(2) us(840) rsadsi(113549) pkcs(1) pkcs-9(9)
111	    smime(16) ct(1) TBD2 }

113	4.  Security Considerations

115	   The security considerations in [RFC5958] apply here.

117	5.  IANA Considerations

119	   For the private key info content types defined in section Section 3,
120	   IANA is requested to assign an object identifier (OID) for each of
121	   the content types.  The OIDs for the content types should be
122	   alloacted in the "SMI Security for S/MIME CMS Content Type" registry
123	   (1.2.840.113549.1.9.16.1), and the description should be set to id-
124	   ct-privateKeyInfo (TDB1) and id-ct-encrPrivateKeyInfo (TBD2).

126	   For the ASN.1 Module in Section 6, IANA is requested to assign an
127	   object identifier (OID) for the module identifier.  The OID for the
128	   module should be allocated in the "SMI Security for S/MIME Module
129	   Identifier" registry (1.2.840.113549.1.9.16.0), and the Description
130	   for the new OID should be set to "id-mod-pkcs8ContentType".

132	6.  ASN.1 Module

134	   The ASN.1 module in this section builds upon the modules in
135	   [RFC5911].

137	   <CODE BEGINS>
138	   PrivateKeyInfoContentTypes
139	    { iso(1) member-body(2) us(840) rsadsi(113549) pkcs(1)
140	      pkcs-9(9) smime(16) modules(0) id-mod-pkcs8ContentType(TBD0) }

142	   DEFINITIONS IMPLICIT TAGS ::=
143	   BEGIN

145	   -- EXPORTS ALL

147	   IMPORTS

149	   CONTENT-TYPE
150	    FROM CryptographicMessageSyntax-2009 -- in [RFC5911]
151	      { iso(1) member-body(2) us(840) rsadsi(113549) pkcs(1)
152	        pkcs-9(9) smime(16) modules(0) id-mod-cms-2004-02(41) }

154	   PrivateKeyInfo, EncryptedPrivateKeyInfo
155	    FROM AsymmetricKeyPackageModuleV1 -- in [RFC5958]
156	       { iso(1) member-body(2) us(840) rsadsi(113549) pkcs(1)
157	         pkcs-9(9) smime(16) modules(0)
158	         id-mod-asymmetricKeyPkgV1(50) }  ;

160	   PrivateKeyInfoContentTypes CONTENT-TYPE ::= {
161	    ct-privateKeyInfo | ct-encrPrivateKeyInfo,
162	    ... -- Expect additional content types --  }

164	   ct-privateKeyInfo CONTENT-TYPE ::= { PrivateKeyInfo
165	    IDENTIFIED BY id-ct-privateKeyInfo }

167	   id-ct-privateKeyInfo OBJECT IDENTIFIER ::= { iso(1)
168	    member-body(2) us(840) rsadsi(113549) pkcs(1) pkcs-9(9)
169	    smime(16) ct(1) TBD1 }

171	   ct-encrPrivateKeyInfo CONTENT-TYPE ::= { EncryptedPrivateKeyInfo
172	    IDENTIFIED BY id-ct-encrPrivateKeyInfo }

174	   id-ct-encrPrivateKeyInfo OBJECT IDENTIFIER ::= { iso(1)
175	    member-body(2) us(840) rsadsi(113549) pkcs(1) pkcs-9(9)
176	    smime(16) ct(1) TBD2 }

178	   END
179	   <CODE ENDS>
