# Document Shepherd Write-Up for <a href="/doc/draft-ietf-spice-glue-id/03/">draft-ietf-spice-glue-id-03</a>

This draft provides a projection from multiple identifier spaces into the URN
space, primarily for identifiers that identify actors in trade.

## Document History

This document has been relatively uncontroversial.  There's a general
recognition in the WG of the importance of addressing this tiny hole in the
problem space and that's exactly what SPICE was chartered to do.  The document
has strong support in the group.  It's a pretty small group, which is really
the only caveat on the strength of consensus.

We have no information on deployments; this is likely an area where deployment
will follow specification.  Risk-wise, that seems acceptable.

## Additional Reviews

The group has not sought URI scheme review.  This is not required, but some of
the folks who might provide that review could have useful input.  Chairs don't
think this is necessary, but we'll note that here for the IESG to consider.
(<a href="/doc/rfc7595/">RFC 7595</a> doesn't establish a mailing list for review like what exists for
media types in <a href="/doc/rfc6838/">RFC 6838</a>, which might be worth considering.)

## Document Shepherd Checks

The document is pretty clean overall.  There were a couple of nits in -03, but
these are minor enough to ignore.

No need for special review, modulo the aforementioned URI review..

I have raised an issue with the editors about having normative references to
the specifications of different identifiers.  I believe that one of these will
be paywalled (we can curse ISO again, I suppose, but it's an important
specification; it might be OK to add a secondary reference to
<a href="https://www.iana.org/assignments/urn-formal/lei">https://www.iana.org/assignments/urn-formal/lei</a> though).
